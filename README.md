# USA Car Crashes Analysis



This repository contains a data-driven project focused on analyzing traffic accidents in the United States. The study uses a large-scale dataset with over 100,000 records to identify how external factors like weather, time of day, speed limits, and driver age influence crash frequency and severity.



The project was completed as a collaborative work by Kateryna Ventskovska and Olha Humenna.



## Project datasets and structure

The analysis was performed using Python in a Jupyter Notebook environment. The workflow covers extensive data preprocessing, managing missing parameters, and statistical visualization.



* `Car_crashes_USA.ipynb` - the complete Jupyter Notebook containing data cleaning, feature restructuring, and data visualizations.



## Key findings

- Surprisingly, roads with the lowest speed limits (10–25 mph) showed the highest average number of vehicles involved per crash. This happens because low speed limits are typical for dense urban areas, intersections, and parking lots, where traffic density leads to multi-car chain reactions.



- While the absolute number of crashes and injuries peaks during clear, dry weather (when road usage is highest), a customized heatmap revealed that the mortality rate relative to total injuries increases significantly during ice and snow conditions. Bad weather causes fewer total accidents, but they are far more severe.



- The data confirms that seatbelts are highly effective. In the vast majority of analyzed crashes, belted passengers avoided fatal or severe injuries, providing a strong argument for seatbelt usage.



- Traffic accidents show a distinct time dependency, peaking between 16:00 and 17:00 (4:00 PM – 5:00 PM) on weekends (Saturday and Sunday), reflecting changing driving patterns during leisure hours.
